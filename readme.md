# ASGUARD01 - SYSTEM STANDARDS

Welcome to the central repository of **ASGUARD01 Corporation's** development standards and architectural guidelines. This repository is organized into distinct categories to facilitate high-quality enterprise software engineering.

## 📁 Repository Structure

### 1. Software Architecture Patterns
*Location: `guides_software_architecture_patterns/`*
Focuses on high-level system design, enterprise patterns, and theoretical foundations of clean code.
- **DESIGN_SOFTWARE_GUIDE_ASGUARD01.md**: Comprehensive guide on SOLID, GoF design patterns, and Clean Architecture for the entire organization.
- **sys_standarts_full_guide.md**: The "Architect's Bible" covering full system standards, distributed patterns, and core code integrity laws.

### 2. Frontend Implementation Methodology
*Location: `guides_frontend_implementation_methodology/`*
Practical standards for building robust, isolated, and state-driven UI modules using Vanilla HTML/CSS/JS.
- **good-style-methodology.md**: Rules for production-grade frontend development focusing on isolation and explicit contracts.
- **containerization-guide.md**: Standards for building self-contained UI widgets.
- **containerization-guide-2.md**: Advanced techniques for module isolation and state management.

### 3. Bootstrap Ecosystem Standards
*Location: `guides_bootstrap_ecosystem_standards/`*
Best practices for integrating and customizing the Bootstrap 5.x framework while avoiding technical debt.
- **bootstrap-scalable-templates-guide.md**: How to build adaptive templates that scale across large projects.
- **bootstrap-unified-guide.md**: Foundation for unified HTML structure across the corporate ecosystem.
- **bootstrap-unified-guide-extended.md**: Extended patterns for complex Bootstrap layouts.
- **bootstrapification-cleaning-guide.md**: Protocol for removing "dirty" custom CSS in favor of Bootstrap utilities.
- **BOOTSTRAPIFICATION_GUIDE.md**: Detailed instructions for migrating legacy pages to Bootstrap standards.
- **kill-fucked-customization-make-clean-bootstrap.md**: Aggressive manual for eliminating bad CSS hacks and adhering to framework-first design.

### 4. Systematic Refactoring Strategies
*Location: `guides_systematic_refactoring_strategies/`*
Structured approaches to improving existing codebases without breaking production functionality.
- **css_refactoring_id_to_class_guide.md**: Strategy for migrating from ID-based selectors to reusable CSS classes.
- **refactoring_guide.md**: Enterprise-level guide for large-scale Bootstrap 5 refactoring.
- **refactoring_plan_guide.md**: Practical templates and checklists for executing refactoring sprints.

### 5. Performance and Security
*Location: `guides_performance_and_security/`*
Guidelines for creating lightning-fast applications and protecting against client-side vulnerabilities.
- **js-performance optimization-base-guide.md**: Essential techniques for DOM optimization and memory management.
- **js-performance optimization-pro-guide.md**: Professional patterns to eliminate browser performance violations and layout thrashing.
- **js-security-guide.md**: Defense-in-depth strategies covering XSS, data validation, and secure JS authoring.

### 6. Visual UX Guidelines
*Location: `guides_visual_ux_guidelines/`*
Standardizing the look and feel of products through design theory and user experience metrics.
- **UI-UX-Web-Development-Guide.md**: Modern UI/UX practices for buttons, states, typography, and interaction design.
- **web-color-guide.md**: Advanced color theory, palette selection, and accessibility-compliant color schemes.

### 7. Accessibility and SEO
*Location: `guides_accessibility_and_seo/`*
Ensuring products are discoverable by search engines and usable by everyone, including people with disabilities.
- **accessibility_guide.md**: Full guide on ARIA roles, A11Y testing, and inclusive design standards.
- **complete-seo-guide.md**: Technical and on-page SEO optimization protocol for modern web applications.

### 8. Documentation and Compliance
*Location: `guides_documentation_and_compliance/`*
Standards for internal code documentation and meeting external marketplace requirements.
- **COMMENTS_GUIDE.md**: Best practices for writing meaningful, intention-based comments.
- **COMMENTS_GUIDE_EXTENDED.md**: Deep dive into documenting complex business logic.
- **compact_comments_guide_html_css_js.md**: Quick reference for shorthand comment formats (TODO, FIXME, PERF).
- **envato-compliance-guide.md**: Strict checklist for preparing code for marketplace submission (privacy, licensing, cleanup).

### 9. Project Metadata
*Location: `guides_project_metadata_index/`*
Contains the foundational project descriptions.
- **readme.md**: Original project entry point.
