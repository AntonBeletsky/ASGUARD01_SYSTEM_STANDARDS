Enterprise Templating Guide

Vanilla JS UI Templating
for Enterprise Applications

A unified, framework-free approach to dynamic HTML rendering — scalable, secure, and maintainable.

Approach innerHTML + Controller Class
Dependencies Zero
Browser Support ES2017+ / All Modern
Pairs With Bootstrap · Tailwind · Custom CSS

This guide defines the standard pattern for building dynamic, data-driven UI components using vanilla JavaScript in enterprise frontends. It is based on a class-based Controller pattern that unifies how rendering, state, events, and escaping are handled across all feature areas — from order lists and user directories to dashboards and notification feeds.

Every example in this guide follows the same architectural skeleton. Learning it once means you can read, maintain, and extend any component in the codebase without context-switching between frameworks or paradigms.

Scope
This guide covers UI components rendered client-side from JavaScript data (API responses, state objects). It does not cover server-side rendering, React/Vue components, or static HTML pages.

01 Core Philosophy

The pattern in this codebase is built on five principles:

1 · Class per feature area Each UI section (orders, users, notifications…) has one Controller class responsible for its DOM subtree. No globals, no loose functions.
2 · innerHTML is the render engine Private methods build HTML strings. One final innerHTML assignment per render cycle. No incremental DOM patching.
3 · Escape everything A dedicated _esc() utility wraps every user-controlled value before it enters an HTML string. No exceptions.
4 · Scoped event delegation All event listeners attach to the container root, not to individual elements. Survives re-renders without leaking listeners.
5 · State lives in this.state The component has a single plain object as its source of truth. Mutate state → call _render(). No other path to DOM updates.
6 · Composable render methods Complex views are assembled from small, single-purpose _renderXxx() methods. Each can be tested and modified in isolation.

02 The Base Controller

Every component inherits from (or mirrors) this base structure. Copy it as your starting point for any new feature area.

/**
 * BaseController — canonical skeleton for all UI controllers.
 * Copy, rename, and fill in the methods marked TODO.
 */
class FeatureController {

  /* ─── STATIC CONFIG ──────────────────────────────────────── */
  // Put lookup maps, label maps, badge configs here.
  static STATUS_MAP = {
    active:   { badge: 'text-bg-success', label: 'Active' },
    inactive: { badge: 'text-bg-secondary', label: 'Inactive' },
  };

  /* ─── CONSTRUCTOR ────────────────────────────────────────── */
  constructor(containerSelector) {
    this.container = document.querySelector(containerSelector);
    if (!this.container)
      throw new Error(`FeatureController: "${containerSelector}" not found`);

    // UI sub-nodes
    this.listEl  = this.container.querySelector('#list');
    this.skelEl  = this.container.querySelector('#skeleton');

    // Single source of truth
    this.state = { filter: 'all', search: '', page: 1 };

    // Timers / abort controllers
    this._searchDebounce = null;

    this._bindEvents();
    this._loadData();
  }

  /* ─── DATA LOADING ───────────────────────────────────────── */
  async _loadData() {
    // TODO: fetch from API, then call _render()
    const data = await fetchSomething();
    this.data = data;
    if (this.skelEl) { this.skelEl.remove(); this.skelEl = null; }
    this._render();
  }

  /* ─── UTILS ──────────────────────────────────────────────── */
  _esc(s) {
    if (typeof s !== 'string') return String(s ?? '');
    return s
      .replace(/&/g, '&amp;')
      .replace(/</g, '&lt;')
      .replace(/>/g, '&gt;')
      .replace(/"/g, '&quot;');
  }

  /* ─── FILTERING ──────────────────────────────────────────── */
  _getFiltered() {
    let list = this.data ?? [];
    if (this.state.filter !== 'all')
      list = list.filter(item => item.status === this.state.filter);
    const q = this.state.search.trim().toLowerCase();
    if (q) list = list.filter(item => item.name.toLowerCase().includes(q));
    return list;
  }

  /* ─── RENDER ─────────────────────────────────────────────── */
  _render() {
    const items = this._getFiltered();
    this.listEl.innerHTML = items.length
      ? items.map(item => this._renderItem(item)).join('')
      : this._renderEmpty();
  }

  /* ─── ITEM TEMPLATE ──────────────────────────────────────── */
  _renderItem(item) {
    // TODO: return an HTML string for a single item
    return `<div class="item">${this._esc(item.name)}</div>`;
  }

  /* ─── EMPTY STATE ────────────────────────────────────────── */
  _renderEmpty() {
    return `<div class="empty-state">No items found.</div>`;
  }

  /* ─── EVENTS ─────────────────────────────────────────────── */
  _bindEvents() {
    // Filter tabs
    this.container.querySelector('[data-filter-group]')
      ?.addEventListener('click', e => {
        const btn = e.target.closest('[data-filter]');
        if (!btn) return;
        this.state.filter = btn.dataset.filter;
        this._render();
      });

    // Search with debounce
    this.container.querySelector('[data-search]')
      ?.addEventListener('input', e => {
        clearTimeout(this._searchDebounce);
        this._searchDebounce = setTimeout(() => {
          this.state.search = e.target.value;
          this._render();
        }, 280);
      });

    // Delegated click on list items
    this.listEl?.addEventListener('click', e => {
      const row = e.target.closest('[data-id]');
      if (!row) return;
      this._onItemClick(row.dataset.id);
    });
  }

  _onItemClick(id) { /* TODO */ }
}

03 Escaping & Security

The _esc() method is the single most important utility in the pattern. It must wrap every value from outside your control before it enters an HTML string — API responses, URL params, user input, translated labels, anything.

// The canonical _esc() implementation
_esc(s) {
  if (typeof s !== 'string') return String(s ?? '');
  return s
    .replace(/&/g,  '&amp;')
    .replace(/</g,  '&lt;')
    .replace(/>/g,  '&gt;')
    .replace(/"/g,  '&quot;');
}
ContextUseExample
Text content_esc(value)${this._esc(user.name)}
HTML attribute_esc(value)title="${this._esc(item.desc)}"
URL in href/src_esc(value) + validate schemeSee URL section below
Known-safe HTMLRaw insert, no _escYour own render methods only
Numbers_esc() handles via String()${this._esc(item.count)}

_safeUrl() — URL validation

For href and src attributes sourced from data, use a dedicated URL validator to prevent javascript: injection:

_safeUrl(url, fallback = '#') {
  try {
    const u = new URL(url, location.origin);
    return ['https:', 'http:', '/'].some(p => u.href.startsWith(p))
      ? this._esc(url)
      : fallback;
  } catch { return fallback; }
}
Never do this
element.innerHTML = userInput — even a single unescaped value from an API response can be a stored XSS vector. Always use _esc().

04 The Render Pipeline

The render pipeline is always one-directional: state → HTML string → innerHTML. Never mutate the DOM directly after rendering.

// Render pipeline: filter → compose → assign
_render() {
  const items = this._getFiltered();

  this.listEl.innerHTML = items.length
    ? items.map(o => this._renderCard(o)).join('')
    : this._renderEmpty(this.state.filter);

  // Post-render hooks: animations, focus management, etc.
  this._afterRender();
}

_afterRender() {
  // Trigger CSS transitions that require a frame boundary
  this.listEl.querySelectorAll('[data-animate]').forEach(el => {
    requestAnimationFrame(() =>
      requestAnimationFrame(() => el.classList.add('animated'))
    );
  });
}

Composing complex cards

Break complex HTML into small private methods. Each method returns a string. The top-level _renderCard() assembles them:

_renderCard(item) {
  return `
  <div class="card" data-id="${this._esc(item.id)}">
    ${this._renderCardHeader(item)}
    <div class="card-body collapse" id="body-${this._esc(item.id)}">
      ${this._renderCardMeta(item)}
      ${this._renderCardActions(item)}
    </div>
  </div>`;
}

_renderCardHeader(item) { return `<div class="card-header">...</div>`; }
_renderCardMeta(item)   { return `<div class="card-meta">...</div>`; }
_renderCardActions(item){ return `<div class="card-actions">...</div>`; }

05 State Management

State is a plain object on this.state. Every user interaction modifies state and triggers a re-render. No direct DOM mutation outside of _render().

// Initial state — always define the full shape upfront
this.state = {
  filter:   'all',
  search:   '',
  page:     1,
  pageSize: 25,
  sort:     { field: 'createdAt', dir: 'desc' },
  selected: new Set(),
};

// Mutating state — always go through a setter that calls _render
_setState(patch) {
  Object.assign(this.state, patch);
  this._render();
}

// Usage in event handlers:
btn.addEventListener('click', () =>
  this._setState({ filter: 'active', page: 1 })
);
Good practice
Always reset page to 1 when changing filter or search. Add this to _setState() automatically if any of those keys are in the patch.

06 Event Binding

Bind all events in _bindEvents(), called once from the constructor. Use event delegation — attach to the container, not to individual rendered elements. This survives re-renders without memory leaks.

_bindEvents() {

  // ── Filter pills ────────────────────────────────────────── */
  this.container.querySelector('#filter-tabs')
    .addEventListener('click', e => {
      const btn = e.target.closest('[data-filter]');
      if (!btn) return;

      // Update UI state
      this.container.querySelectorAll('[data-filter]').forEach(b => {
        b.classList.toggle('active', b === btn);
        b.setAttribute('aria-selected', b === btn);
      });

      this._setState({ filter: btn.dataset.filter, page: 1 });
    });

  // ── Debounced search ─────────────────────────────────────── */
  this.container.querySelector('#search-input')
    .addEventListener('input', e => {
      clearTimeout(this._searchDebounce);
      this._searchDebounce = setTimeout(() =>
        this._setState({ search: e.target.value, page: 1 }),
        280
      );
    });

  // ── Delegated click on dynamic list items ────────────────── */
  this.listEl.addEventListener('click', e => {

    // "Delete" action button
    const delBtn = e.target.closest('[data-action="delete"]');
    if (delBtn) { this._onDelete(delBtn.dataset.id); return; }

    // Row expand / collapse
    const row = e.target.closest('[data-expand]');
    if (row) { this._onExpand(row.dataset.id); return; }
  });

  // ── Keyboard: close modal on Escape ─────────────────────── */
  document.addEventListener('keydown', e => {
    if (e.key === 'Escape') this._closeModal();
  });
}

07 Use Case: Orders List Reference

The canonical example. Full pattern with status stepper, tracking history accordion, and filtering.

class OrdersController {

  static STATUS_CFG = {
    processing: { badge: 'text-bg-info',      label: 'Processing' },
    shipped:    { badge: 'text-bg-warning',   label: 'Shipped'    },
    delivered:  { badge: 'text-bg-success',   label: 'Delivered'  },
    cancelled:  { badge: 'text-bg-danger',    label: 'Cancelled'  },
  };

  constructor(sel) {
    this.container = document.querySelector(sel);
    this.listEl = this.container.querySelector('#orders-list');
    this.state  = { filter: 'all', search: '' };
    this._bindEvents();
    setTimeout(() => { this.data = ORDERS; this._render(); }, 500);
  }

  _esc(s) { /* ... standard impl */ }

  _fmtDate(s) {
    return s ? new Date(s).toLocaleDateString('en-US', {
      year: 'numeric', month: 'long', day: 'numeric'
    }) : '—';
  }

  _fmtCur(v, c = 'USD') {
    return new Intl.NumberFormat('en-US', {
      style: 'currency', currency: c
    }).format(v);
  }

  _getFiltered() {
    let list = this.data ?? [];
    if (this.state.filter !== 'all')
      list = list.filter(o => o.status === this.state.filter);
    const q = this.state.search.trim().toLowerCase();
    if (q) list = list.filter(o => o.id.toLowerCase().includes(q));
    return list;
  }

  _render() {
    const list = this._getFiltered();
    this.listEl.innerHTML = list.length
      ? list.map(o => this._renderOrder(o)).join('')
      : this._renderEmpty();
  }

  _renderOrder(o) {
    const { badge, label } = OrdersController.STATUS_CFG[o.status] ?? {};
    return `
    <div class="order-card border mb-3" data-id="${this._esc(o.id)}">
      <div class="d-flex justify-content-between p-3">
        <span class="fw-semibold">${this._esc(o.id)}</span>
        <span class="badge ${this._esc(badge)}">${this._esc(label)}</span>
      </div>
      <div class="collapse" id="ord-${this._esc(o.id)}">
        ${this._renderStepper(o.steps)}
        ${o.items.map(i => this._renderProduct(i)).join('')}
      </div>
    </div>`;
  }
  /* ... _renderStepper, _renderProduct, _renderEmpty etc */
}

08 Use Case: User Directory Core

Avatar initials, role badges, status dots, inline actions. Filterable by department and role.

class UserDirectoryController {

  static ROLE_COLORS = {
    admin:     'text-bg-danger',
    manager:   'text-bg-warning',
    developer: 'text-bg-info',
    viewer:    'text-bg-secondary',
  };

  // Generate a deterministic pastel avatar color from name
  _avatarColor(name) {
    const colors = [
      '#4f8ef7','#7ee8a2','#f7c948','#f76e6e','#c792ea','#89ddff'
    ];
    const hash = [...name].reduce((a, c) => a + c.charCodeAt(0), 0);
    return colors[hash % colors.length];
  }

  _initials(name) {
    return name.split(' ').map(w => w[0]).slice(0, 2).join('').toUpperCase();
  }

  _renderUser(u) {
    const roleBadge = UserDirectoryController.ROLE_COLORS[u.role] ?? 'text-bg-secondary';
    const avatarBg  = this._avatarColor(u.name);
    const initials  = this._initials(u.name);

    return `
    <div class="user-row d-flex align-items-center gap-3 p-3 border-bottom" data-id="${this._esc(u.id)}">

      <div class="avatar rounded-circle d-flex align-items-center justify-content-center flex-shrink-0"
           style="width:40px;height:40px;background:${avatarBg};font-weight:600;color:#fff;font-size:.85rem">
        ${initials}
      </div>

      <div class="flex-grow-1 mw-0">
        <div class="fw-semibold text-truncate">${this._esc(u.name)}</div>
        <div class="text-secondary small text-truncate">${this._esc(u.email)}</div>
      </div>

      <div class="d-none d-md-block text-secondary small">${this._esc(u.department)}</div>

      <span class="badge ${this._esc(roleBadge)}">${this._esc(u.role)}</span>

      <div class="status-dot rounded-circle flex-shrink-0"
           style="width:9px;height:9px;background:${u.online ? '#7ee8a2' : '#555'}"

        View
      </button>
    </div>`;
  }
}

09 Use Case: Dashboard KPI Cards Core

Metric cards with trend indicators, sparklines via inline SVG, and delta formatting.

class DashboardController {

  _fmtDelta(delta) {
    const sign  = delta >= 0 ? '+' : '';
    const cls   = delta >= 0 ? 'text-success' : 'text-danger';
    const arrow = delta >= 0 ? '↑' : '↓';
    return `<span class="${cls}">${arrow} ${sign}${delta.toFixed(1)}%</span>`;
  }

  _sparklinePath(values) {
    const w = 80, h = 32;
    const min = Math.min(...values), max = Math.max(...values);
    const range = max - min || 1;
    const pts = values.map((v, i) => {
      const x = (i / (values.length - 1)) * w;
      const y = h - ((v - min) / range) * h;
      return `${x.toFixed(1)},${y.toFixed(1)}`;
    });
    return `M ${pts.join(' L ')}`;
  }

  _renderKpiCard(metric) {
    const path = this._sparklinePath(metric.trend);
    return `
    <div class="col-6 col-lg-3">
      <div class="card h-100 p-3">
        <div class="text-secondary small mb-1">${this._esc(metric.label)}</div>
        <div class="d-flex align-items-end justify-content-between gap-2">
          <div>
            <div class="fs-4 fw-bold">${this._esc(metric.value)}</div>
            <div class="small mt-1">${this._fmtDelta(metric.delta)} vs last period</div>
          </div>
          <svg width="80" height="32" viewBox="0 0 80 32" fill="none"
               aria-hidden="true">
            <path d="${path}"
                  stroke="${metric.delta >= 0 ? '#7ee8a2' : '#f76e6e'}"
                  stroke-width="2" fill="none" stroke-linecap="round"/>
          </svg>
        </div>
      </div>
    </div>`;
  }

  _render() {
    this.gridEl.innerHTML = `
      <div class="row g-3">
        ${this.metrics.map(m => this._renderKpiCard(m)).join('')}
      </div>`;
  }
}

10 Use Case: Dynamic Forms Advanced

Forms built from a schema — field type dispatching, validation state rendering, conditional field visibility.

class DynamicFormController {

  // Field type → render method dispatch table
  static FIELD_RENDERERS = {
    text:     '_renderTextField',
    email:    '_renderTextField',
    number:   '_renderTextField',
    select:   '_renderSelectField',
    textarea: '_renderTextareaField',
    checkbox: '_renderCheckboxField',
    radio:    '_renderRadioGroup',
  };

  _renderField(field, errors = {}) {
    const method = DynamicFormController.FIELD_RENDERERS[field.type];
    if (!method || typeof this[method] !== 'function') return '';
    const error = errors[field.name];
    return this[method](field, error);
  }

  _fieldWrapper(field, inputHtml, error) {
    return `
    <div class="mb-3" data-field="${this._esc(field.name)}">
      <label class="form-label fw-semibold" for="f-${this._esc(field.name)}">
        ${this._esc(field.label)}
        ${field.required ? '<span class="text-danger">*</span>' : ''}
      </label>
      ${inputHtml}
      ${error ? `<div class="invalid-feedback d-block">${this._esc(error)}</div>` : ''}
      ${field.hint ? `<div class="form-text">${this._esc(field.hint)}</div>` : ''}
    </div>`;
  }

  _renderTextField(field, error) {
    const cls = error ? 'form-control is-invalid' : 'form-control';
    const input = `
      <input type="${this._esc(field.type)}"
             id="f-${this._esc(field.name)}"
             name="${this._esc(field.name)}"
             class="${cls}"
             placeholder="${this._esc(field.placeholder ?? '')}"
             ${field.required ? 'required' : ''}
             ${field.maxlength ? `maxlength="${field.maxlength}"` : ''}
             value="${this._esc(field.value ?? '')}">`;
    return this._fieldWrapper(field, input, error);
  }

  _renderSelectField(field, error) {
    const cls = error ? 'form-select is-invalid' : 'form-select';
    const opts = (field.options ?? []).map(o =>
      `<option value="${this._esc(o.value)}"
      ${o.value === field.value ? 'selected' : ''}>
      ${this._esc(o.label)}</option>`
    ).join('');
    return this._fieldWrapper(field,
      `<select id="f-${this._esc(field.name)}" name="${this._esc(field.name)}"
               class="${cls}">${opts}</select>`, error);
  }
}

11 Use Case: Notification Feed Core

Toast queue + persistent notification list. Icon dispatch by type, relative time formatting, read/unread state.

class NotificationsController {

  static TYPE_CFG = {
    success: { icon: 'check_circle',  cls: 'text-success' },
    error:   { icon: 'error',          cls: 'text-danger'  },
    warning: { icon: 'warning',        cls: 'text-warning' },
    info:    { icon: 'info',           cls: 'text-info'    },
  };

  _relTime(isoStr) {
    const diff = (Date.now() - new Date(isoStr)) / 1000;
    if (diff < 60)  return 'just now';
    if (diff < 3600) return `${Math.floor(diff/60)}m ago`;
    if (diff < 86400) return `${Math.floor(diff/3600)}h ago`;
    return `${Math.floor(diff/86400)}d ago`;
  }

  _renderNotification(n) {
    const { icon, cls } = NotificationsController.TYPE_CFG[n.type] ?? NotificationsController.TYPE_CFG.info;
    return `
    <div class="notif-row d-flex gap-3 p-3 ${n.read ? '' : 'bg-primary-subtle'}"
         data-id="${this._esc(n.id)}" role="listitem">
      <span class="material-symbols-rounded ${cls} flex-shrink-0">${icon}</span>
      <div class="flex-grow-1 mw-0">
        <div class="small fw-semibold">${this._esc(n.title)}</div>
        <div class="small text-secondary text-truncate">${this._esc(n.body)}</div>
      </div>
      <div class="text-secondary" style="font-size:.7rem;white-space:nowrap">
        ${this._esc(this._relTime(n.timestamp))}
      </div>
      ${!n.read ? `<div style="width:8px;height:8px;border-radius:50%;background:var(--bs-primary);flex-shrink:0;margin-top:6px"></div>` : ''}
    </div>`;
  }

  // Toast queue — separate from the persistent list
  showToast(type, title, body, durationMs = 4000) {
    const { icon, cls } = NotificationsController.TYPE_CFG[type] ?? {};
    const id = `toast-${Date.now()}`;
    const el = document.createElement('div');
    el.innerHTML = `
      <div id="${id}" class="toast align-items-center border-0 show" role="alert">
        <div class="d-flex">
          <div class="toast-body d-flex gap-2">
            <span class="material-symbols-rounded ${cls}">${icon}</span>
            <div>
              <strong>${this._esc(title)}</strong>
              <div class="small text-secondary">${this._esc(body)}</div>
            </div>
          </div>
        </div>
      </div>`;
    this.toastContainer.appendChild(el.firstElementChild);
    setTimeout(() => document.getElementById(id)?.remove(), durationMs);
  }
}

12 Use Case: Data Tables Advanced

Sortable columns, row selection, bulk actions, inline cell formatters by column type.

class DataTableController {

  constructor(sel, columns) {
    this.columns = columns; // column schema array
    this.state   = { sort: { col: columns[0].key, dir: 'asc' }, selected: new Set() };
    /* ... */
  }

  _getSorted(data) {
    const { col, dir } = this.state.sort;
    return [...data].sort((a, b) => {
      const av = a[col] ?? '', bv = b[col] ?? '';
      const cmp = typeof av === 'number'
        ? av - bv
        : String(av).localeCompare(String(bv));
      return dir === 'asc' ? cmp : -cmp;
    });
  }

  _renderHead() {
    return `<thead><tr>
      <th><input type="checkbox" id="select-all"></th>
      ${this.columns.map(c => {
        const active = this.state.sort.col === c.key;
        const dir    = this.state.sort.dir;
        return `<th data-sort="${this._esc(c.key)}" style="cursor:pointer">
          ${this._esc(c.label)}
          ${active ? (dir === 'asc' ? '↑' : '↓') : '⇅'}
        </th>`;
      }).join('')}
    </tr></thead>`;
  }

  _renderCell(col, value, row) {
    switch (col.type) {
      case 'currency':
        return `<td class="text-end font-monospace">
                 ${this._esc(this._fmtCur(value))}</td>`;
      case 'date':
        return `<td>${this._esc(this._fmtDate(value))}</td>`;
      case 'badge':
        const cfg = (col.map ?? {})[value] ?? { cls: 'text-bg-secondary', label: value };
        return `<td><span class="badge ${this._esc(cfg.cls)}">${this._esc(cfg.label)}</span></td>`;
      case 'link':
        return `<td><a href="${this._safeUrl(row[col.urlKey])}">${this._esc(value)}</a></td>`;
      default:
        return `<td>${this._esc(value)}</td>`;
    }
  }

  _renderRow(row) {
    const checked = this.state.selected.has(row.id) ? 'checked' : '';
    return `
    <tr data-id="${this._esc(row.id)}" class="${checked ? 'table-active' : ''}">
      <td><input type="checkbox" data-select="${this._esc(row.id)}" ${checked}></td>
      ${this.columns.map(c => this._renderCell(c, row[c.key], row)).join('')}
    </tr>`;
  }
}

13 Use Case: Activity Feed Core

Timeline grouped by day, actor avatars, action verbs, polymorphic payload rendering.

class ActivityFeedController {

  _groupByDay(events) {
    return events.reduce((map, e) => {
      const day = e.timestamp.slice(0, 10); // YYYY-MM-DD
      (map[day] ??= []).push(e);
      return map;
    }, {});
  }

  _renderPayload(event) {
    switch (event.type) {
      case 'comment':
        return `<blockquote class="border-start ps-2 text-secondary small mt-1">
                 ${this._esc(event.payload.text)}</blockquote>`;
      case 'status_change':
        return `<span class="badge text-bg-secondary">${this._esc(event.payload.from)}</span>
                 → <span class="badge text-bg-success">${this._esc(event.payload.to)}</span>`;
      case 'file_upload':
        return `<a href="${this._safeUrl(event.payload.url)}" class="small">
                 📎 ${this._esc(event.payload.filename)}</a>`;
      default:
        return '';
    }
  }

  _renderEvent(e) {
    return `
    <div class="d-flex gap-2 py-2">
      <div class="avatar-sm rounded-circle bg-secondary d-flex align-items-center justify-content-center
                  text-white flex-shrink-0" style="width:32px;height:32px;font-size:.75rem">
        ${this._initials(e.actor.name)}
      </div>
      <div class="flex-grow-1">
        <div class="small">
          <strong>${this._esc(e.actor.name)}</strong>
          ${this._esc(e.verb)}
          <span class="text-secondary">${this._relTime(e.timestamp)}</span>
        </div>
        ${this._renderPayload(e)}
      </div>
    </div>`;
  }

  _render() {
    const groups = this._groupByDay(this.events);
    this.feedEl.innerHTML = Object.entries(groups).map(([day, events]) => `
      <div class="mb-4">
        <div class="text-secondary small fw-bold text-uppercase mb-2">
          ${this._esc(this._fmtDate(day))}
        </div>
        <div class="border-start ms-2 ps-3">
          ${events.map(e => this._renderEvent(e)).join('')}
        </div>
      </div>`).join('');
  }
}

14 Use Case: Modals Core

Render modal content from data, manage open/close lifecycle, restore focus on dismiss.

class ModalController {

  constructor() {
    this.overlay  = document.querySelector('#modal-overlay');
    this.modalBox = document.querySelector('#modal-box');
    this._trigger = null; // element that opened the modal (restore focus)
    this._bindEvents();
  }

  open(data, triggerEl) {
    this._trigger = triggerEl ?? document.activeElement;
    this.modalBox.innerHTML = this._renderModal(data);
    this.overlay.classList.add('show');
    document.body.style.overflow = 'hidden';

    // Move focus into modal (accessibility)
    const firstFocusable = this.modalBox.querySelector(
      'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
    );
    firstFocusable?.focus();
  }

  close() {
    this.overlay.classList.remove('show');
    document.body.style.overflow = '';
    this._trigger?.focus(); // restore focus
    this._trigger = null;
  }

  _renderModal(data) {
    return `
    <div class="modal-header">
      <h5 class="modal-title">${this._esc(data.title)}</h5>
      <button type="button" class="btn-close" data-action="close-modal"
              aria-label="Close"></button>
    </div>
    <div class="modal-body">${this._renderModalBody(data)}</div>
    <div class="modal-footer">
      <button class="btn btn-secondary" data-action="close-modal">Cancel</button>
      <button class="btn btn-primary" data-action="confirm-modal"
              data-id="${this._esc(data.id)}">Confirm</button>
    </div>`;
  }

  _bindEvents() {
    this.overlay.addEventListener('click', e => {
      if (e.target === this.overlay) this.close(); // backdrop click
      if (e.target.closest('[data-action="close-modal"]')) this.close();
    });
    document.addEventListener('keydown', e => {
      if (e.key === 'Escape' && this.overlay.classList.contains('show'))
        this.close();
    });
  }
}

15 Pagination

// Add to state: { page: 1, pageSize: 25 }

_paginate(list) {
  const { page, pageSize } = this.state;
  const start = (page - 1) * pageSize;
  return {
    items:    list.slice(start, start + pageSize),
    total:    list.length,
    pages:    Math.ceil(list.length / pageSize),
    page,
  };
}

_renderPagination({ page, pages, total, pageSize }) {
  if (pages <= 1) return '';
  const nums = Array.from({ length: pages }, (_, i) => i + 1);

  return `
  <nav class="d-flex align-items-center justify-content-between mt-3">
    <div class="text-secondary small">
      ${(page-1)*pageSize+1}–${Math.min(page*pageSize, total)} of ${total}
    </div>
    <ul class="pagination pagination-sm mb-0">
      <li class="page-item ${page === 1 ? 'disabled' : ''}">
        <button class="page-link" data-page="${page - 1}">‹</button></li>
      ${nums.map(n => `
        <li class="page-item ${n === page ? 'active' : ''}">
          <button class="page-link" data-page="${n}">${n}</button>
        </li>`).join('')}
      <li class="page-item ${page === pages ? 'disabled' : ''}">
        <button class="page-link" data-page="${page + 1}">›</button></li>
    </ul>
  </nav>`;
}

18 Performance

When to use DocumentFragment

For lists over ~200 items where individual items need post-render DOM access (e.g. attaching canvas, initializing third-party widgets), prefer DocumentFragment over full innerHTML replacement:

_render() {
  const frag = document.createDocumentFragment();
  for (const item of this._getFiltered()) {
    const wrap = document.createElement('div');
    wrap.innerHTML = this._renderItem(item);
    frag.appendChild(wrap.firstElementChild);
  }
  this.listEl.replaceChildren(frag);
}

Debounce vs Throttle

ScenarioUseDelay
Search inputDebounce250–300ms
Window resizeDebounce150ms
Scroll handlerThrottle100ms
Drag eventsThrottle16ms (rAF)

Avoid layout thrash in post-render hooks

// ❌ Causes layout thrash — reads then writes in a loop
items.forEach(el => {
  const h = el.offsetHeight; // forces reflow
  el.style.marginTop = h / 2 + 'px'; // invalidates layout
});

// ✅ Batch reads first, then writes
const heights = items.map(el => el.offsetHeight); // all reads
items.forEach((el, i) => el.style.marginTop = heights[i] / 2 + 'px');

20 Implementation Checklist

Use before shipping any new controller
Run through this list on every new feature area.

Security

  • Every API-sourced value passes through _esc() before entering HTML
  • URLs use _safeUrl() — no raw href from data without validation
  • No eval(), Function(), or setTimeout(string) anywhere

Architecture

  • One controller class per feature area, scoped to one container element
  • All state lives in this.state — no DOM reads to determine state
  • All events use delegation from container root — no listeners on rendered elements
  • _bindEvents() called exactly once from constructor
  • Static config (badge maps, label maps) defined as static class properties

UX

  • Skeleton screen shown during initial data load
  • Empty state rendered for every filter/search combination that returns 0 results
  • Search input is debounced (250–300ms)
  • Pagination resets to page 1 on filter or search change
  • Modals restore focus to trigger element on close
  • CSS transitions use requestAnimationFrame double-tick for reliability

Accessibility

  • List container has role="list", items have role="listitem"
  • Live region (aria-live="polite") on the list container for screen reader announcements
  • Filter tabs use role="tab" and aria-selected
  • Icon-only buttons have aria-label
  • Status dots and decorative icons have aria-hidden="true"

Code quality

  • 'use strict' at top of every script block
  • All render methods return a string — no side effects
  • No raw number/boolean interpolation without _esc()
  • Complex cards broken into focused sub-render methods
  • No jQuery or framework imports — this pattern is intentionally zero-dependency
