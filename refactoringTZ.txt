Technical Specification
Project Refactoring & Modernization
1. Objective
Преобразовать существующий проект (с устаревшей архитектурой и техническим долгом) в современное, масштабируемое и поддерживаемое решение, сохранив его исходное функциональное назначение и концептуальную идентичность, но полностью обновив техническую реализацию.

2. Input Data
В рамках анализа предоставляются:
Скриншоты интерфейса
Исходный код проекта
Контекст использования (если имеется)
До получения явной команды «делай» выполняется только аналитическая часть без реализации.

3. Stage 1 — System Analysis
3.1 Code Analysis
Архитектура проекта
Структура файлов
Используемые технологии
Зависимости (включая jQuery и сторонние библиотеки)
Технический долг
Дублирование логики
Потенциальные конфликты имён (особенно с Bootstrap)
Соответствие стандартам W3C (HTML, CSS, JS)
3.2 UI Analysis
Структура интерфейса
Иерархия элементов
Консистентность визуальной системы
Соответствие современным UI-паттернам
3.3 UX Analysis
Пользовательские сценарии
Жизненный цикл взаимодействия
Состояния интерфейса (loading, empty, error, success, active, disabled и т.д.)
Переходы между состояниями
Потенциальные UX-проблемы
3.4 Adaptive Behavior
Mobile-first подход
Поведение на смартфонах
Breakpoints
Touch-интеракции
3.5 Accessibility (A11y)
ARIA-атрибуты
Фокус-менеджмент
Клавиатурная навигация
Контрастность
Семантическая корректность HTML
3.6 Documentation Quality
Качество комментариев
Стиль документирования
Соответствие best practices разработки корпоративного ПО

4. Stage 2 — Architectural Redesign (Without Implementation)
Необходимо:
Проанализировать назначение проекта
Определить его доменную модель
Предложить архитектурный подход
Описать миграцию с jQuery на ES6+
Описать интеграцию с Bootstrap 5.3+
Продумать модульную структуру
Изолировать кастомные стили от Bootstrap (контейнеризация, namespace)

5. Future Implementation Requirements (After “делай”)
5.1 Technology Stack
Bootstrap 5.3+
Vanilla JavaScript (ES6+)
Без jQuery
Соответствие W3C стандартам
5.2 Architecture
Чёткое разделение ответственности
Компонентный подход
Изоляция кастомного CSS
Отсутствие конфликтов с Bootstrap
Предсказуемая структура классов
5.3 UI / UX
Современный визуальный стиль
Консистентная дизайн-система
Mobile-first
Корректные состояния интерфейса
Логичная навигация
5.4 Accessibility
Полная поддержка клавиатурной навигации
ARIA-поддержка
Семантическая разметка
Корректная работа focus-state
5.5 Documentation & Comments
Production-grade комментарии
Чёткое описание логических блоков
Архитектурные пояснения
Комментарии по стандартам enterprise-разработки

6. Expected Result
После команды «делай» должна быть создана:
Самодостаточная страница на Bootstrap 5
Реализация на ES6 (без jQuery)
Полностью прокомментированный код (HTML, CSS, JS)
Enterprise-уровень качества
Готовность к масштабированию
