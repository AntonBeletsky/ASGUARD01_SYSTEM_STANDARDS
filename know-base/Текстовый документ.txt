Обдумал структурирование информации без конкретных деталей
Полный pipeline из 4 скриптов. Вот как это работает:

Воркфлоу:

guide.json
    │
    ▼
generate_parts.py  →  parts/part_01_clean-code.md
                       parts/part_02_object-oriented-design.md
                       ...40 файлов...
    │
    ▼
fill_sections.py   →  заполняет каждую секцию через Claude API
                       (одну, том целиком, или всё сразу)
    │
    ▼
combine_parts.py   →  final_guide.md (один файл, глобальный TOC)
Команды:

bash
# 1. Сгенерировать пустые шаблоны
python generate_parts.py guide.json ./parts/

# 2. Проверить прогресс
python status.py ./parts/
python status.py ./parts/ --list-empty --vol 3

# 3. Заполнить (нужен ANTHROPIC_API_KEY)
export ANTHROPIC_API_KEY=sk-...
python fill_sections.py --volume 1           # один том
python fill_sections.py --section 1.1.1      # одна секция
python fill_sections.py                      # всё (481 секция)
python fill_sections.py --dry-run            # проверить без вызовов

# 4. Склеить в финальный файл
python combine_parts.py ./parts/ ./output/final_guide.md
python combine_parts.py --skip-empty         # только заполненные
Ключевые детали:

каждая секция обёрнута в CONTENT_START/END маркеры — fill_sections.py точно попадает в нужное место
status: empty/filled в метаданных — status.py и combine_parts.py видят прогресс
generate_parts.py не перезаписывает уже существующие файлы — безопасно перезапускать
combine_parts.py --skip-empty выдаёт чистый MD только из заполненного
